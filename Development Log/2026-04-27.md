### Step 4 — Worker Cost Reporting

Created `skills/worker--dispatch/SKILL.md` in the repo. Workers are single `messages.create()` API calls dispatched in parallel via `asyncio.gather()`, not full Claude Code sessions.

**Key decisions:**
- Default model: `claude-haiku-4-5-20251001`
- Cost reported per worker call (not aggregated) — enables per-task breakdown in the board dashboard
- `biller: "worker"` + `billingType: "worker_dispatch"` in every cost event to distinguish from direct agent costs
- Cost reporting failures are non-fatal — log warning, continue

### Step 5 — Review Checkpoint

Updated SWE Lead instructions in `references/shannon-company.md` to use Paperclip's native `request_confirmation` interaction instead of a free-text CHECKPOINT comment.

**Why this is better than the comment-based MVP:**
- Board sees a structured approval card (Accept / Reject with reason) — not a comment to read and manually reply to
- `continuationPolicy: "wake_assignee"` wakes SWE Lead automatically after board accepts
- On rejection: board provides a structured reason, SWE Lead reads it and re-runs sandbox
- No server code changes required — uses the existing interactions API

## Key decision today: agentic ≠ procedural

Discovered that `ai-software-factory` has 13 slash commands that map exactly to the skills referenced in AGENTS.md — but none exist yet in `paperclip-shannon/skills/`.

This surfaced an important distinction worth recording:

> Paperclip makes agents capable of acting autonomously. Skills define *how* they act consistently.

Without skills, an agent relies on Claude's general knowledge — output is correct sometimes but non-deterministic. With skills, the procedure is encoded: the agent reads project-specific config, follows a defined template, enforces specific quality gates. Skills are the surgical protocols of the pipeline.

This is documented fully in [[AI Software Factory/Building/Skill Port Analysis — ai-software-factory ke paperclip-shannon]].
