### Step 9 — Workers: Anthropic SDK + WORKER_API_KEY + Sonnet

Workers switched from `claude -p` to direct `anthropic.AsyncAnthropic()` calls using `WORKER_API_KEY` and `claude-sonnet-4-6`.

**Why `WORKER_API_KEY` not `ANTHROPIC_API_KEY`:** Claude Code picks up `ANTHROPIC_API_KEY` and switches the orchestrator away from subscription auth. `WORKER_API_KEY` is ignored by Claude Code — orchestrators stay on Max subscription, workers use the separate API key.

**Why Sonnet not Haiku:** Better output quality for code/design/test tasks. Cost reporting updated to Sonnet pricing ($3/$15 per MTok).

**Key benefit:** The worker key can come from a completely different Anthropic account — Russell's API key for workers, Max subscription for orchestrators.

**Intermediate commit** (`0ec9e693`) used `claude -p` as a temporary fallback during debugging; `9d423414` is the final correct implementation.

### Step 10 — Company Delete Bug Fix

`DELETE /api/companies/:id` returned 500 due to 17 tables with `companyId` FKs missing from the delete transaction.

**Root cause:** `server/src/services/companies.ts` had the original 5-table delete but Paperclip had grown to include 17 more FK tables over time.

**Tables added:** `issueThreadInteractions`, `issueAttachments`, `issueLabels`, `issueRelations`, `issueWorkProducts`, `issueExecutionDecisions`, `issueInboxArchives`, `agentConfigRevisions`, `budgetIncidents`, `budgetPolicies`, `inboxDismissals`, `feedbackVotes`, `workspaceOperations`, `workspaceRuntimeServices`, `executionWorkspaces`, `environments`, `environmentLeases`.

Also fixed delete order: `costEvents` must be deleted before `heartbeatRuns`.

### Bonus fix — NewProjectDialog body scrollable

Long project descriptions and property chips were overflowing the dialog. Added `flex/max-h-[90vh]` to `DialogContent` and wrapped the body in `overflow-y-auto`. Header and footer remain fixed.
